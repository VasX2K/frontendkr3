# Заметки PWA (Практические занятия 13–18)

PWA-приложение для управления заметками и задачами с поддержкой офлайн-режима, push-уведомлений, WebSocket-синхронизации и напоминаний.

## Практика 13 — Service Worker

- Реализована регистрация Service Worker (`sw.js`)
- Кэширование статических ресурсов (HTML, JS, иконки, manifest)
- Стратегия **Cache First** для статики и **Network First** для динамического контента (`/content/*`)
- Данные заметок хранятся в `localStorage`
- Приложение полностью работает офлайн

## Практика 14 — Web App Manifest

- Создан `manifest.json` с полями: `name`, `short_name`, `start_url`, `display`, `background_color`, `theme_color`, `description`, `icons`
- Подготовлен набор PNG-иконок (16×16 — 512×512)
- Манифест подключён в `<head>` через `<link rel="manifest">`
- Добавлены мета-теги для Android и iOS
- Service Worker обновлён для кэширования иконок и манифеста

## Практика 15 — HTTPS + App Shell

- Реализована архитектура **App Shell**: минимальный каркас (`index.html`) + динамическая подгрузка контента через `fetch`
- Страницы: `content/home.html` (заметки) и `content/about.html` (о приложении)
- Навигация через вкладки с загрузкой контента по запросу
- Поддержка запуска через HTTPS (локальный сертификат через `mkcert`)

## Практика 16 — WebSocket + Push

- Сервер на **Express + Socket.IO** (`server.js`)
- Двусторонняя связь в реальном времени: при добавлении заметки событие `newTask` рассылается всем клиентам через `taskAdded`
- Всплывающее уведомление при получении задачи от другого клиента
- **Push-уведомления** через Web Push API с VAPID-ключами
- Кнопки включения/отключения уведомлений в интерфейсе
- Эндпоинты `/subscribe` и `/unsubscribe` для управления подписками

## Практика 17 — Детализация Push (Напоминания)

- Форма создания заметки с напоминанием (текст + дата/время)
- Структура заметок расширена полем `reminder` (timestamp)
- Сервер планирует push-уведомления через `setTimeout`
- Service Worker обрабатывает действие **«Отложить на 5 минут»** через `notificationclick`
- Эндпоинт `/snooze` для переноса напоминания

## Практика 18 — Подготовка к контрольной работе №3

- Протестирована работоспособность приложения
- Подготовлен данный README.md

## Запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Генерация VAPID-ключей

```bash
npx web-push generate-vapid-keys
```

Скопируйте публичный и приватный ключи и вставьте их в `server.js` (переменная `vapidKeys`) и в `app.js` (функция `subscribeToPush`, параметр `applicationServerKey`).

### 3. Запуск сервера

```bash
npm start
```

Сервер запустится на `http://localhost:3001`.

### 4. (Опционально) HTTPS через mkcert

```bash
mkcert -install
mkcert localhost 127.0.0.1 ::1
npx http-server --ssl --cert localhost.pem --key localhost-key.pem -p 3000
```

> **Примечание:** для HTTPS-режима потребуется отдельный запуск `server.js` и раздача статики через `http-server`, т.к. `server.js` использует свой HTTP-сервер.

## Проверка

1. Откройте `http://localhost:3001` в двух вкладках
2. Добавьте заметку — она появится в обеих вкладках (WebSocket)
3. Включите push-уведомления кнопкой — получите системное уведомление
4. Создайте заметку с напоминанием — push придёт в указанное время
5. Отключите сеть — приложение продолжит работать из кэша

## Технологии

- **Service Worker** — кэширование, офлайн-режим, push-уведомления
- **Web App Manifest** — установка как PWA
- **App Shell** — мгновенная загрузка каркаса
- **WebSocket (Socket.IO)** — синхронизация в реальном времени
- **Web Push API** — системные уведомления
- **localStorage** — локальное хранение данных
